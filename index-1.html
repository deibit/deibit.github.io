    Gnome: Gloobus-Preview

    David García

    2010-02-25 18:02

    Una herramienta interesante que tiene MacOSX es ‘Quick Look’, con ella no necesitamos abrir un fichero para ver su contenido, con tan solo pulsar la barra espaciadora nos presentará una vista previa.

    Para Gnome existe Gloobus-Preview, soporta 40 formatos de archivo incluyendo sonido y video.

    El paquete deb para Ubuntu se encuentra aquí.

    Para mayor comodidad, añadimos el repositorio para que se actualice:

    sudo add-apt-repository ppa:gloobus-dev/gloobus-preview

    Actualizamos apt e instalamos Gloobus:

    sudo apt-get install gloobus-preview

    Editar y recargar el .vimrc

    David García

    2010-02-22 16:09

    Si queremos editar y recargar nuestro archivo de configuración de Vim (.vimrc en Linux y compañía, _vimrc en Windows) para efectuar ciclos de prueba y error, hay dos comandos Ex y una varible de entorno que nos van a facilitar esta operación.

    El comando Ex source leera e interpretará el archivo que le pasemos, en este caso será el .vimrc.

    A su vez, la variable $MYVIMRC contiene el path hacia el archivo .vimrc. Puede comprobarse con un :echo $MYVIMRC.

    Si queremos editarlo de manera eventual, tan solo tenemos que abrirlo así:

    :e $MYVIMRC

    y cuando terminemos, volvemos a recargar el archivo para que nuestros cambios surtan efecto:

    :source $MYVIMRC

    Lo interesante es -si le metemos mano al vimrc bastante a menudo- mapear esas dos operaciones hacia cualquier combinación de teclas que tengamos libre añadiendo al vimrc las siguientes lineas:

    map s :source $MYVIMRC
    map e :e $MYVIMRC

    Más plugins para vim

    David García

    2010-02-21 07:57

    SuperTab: Hay dos, el original y otro que continua con el mantenimiento, ya que el plugin original está descontinuado. Este plugin emplea la tecla para funciones de autocompletado, adaptandose al sistema de completado que estemos usando.

    snipMate: Copia una función muy (muy muy muy) útil del editor por excelencia del mundo Mac, el TextMate. Con el podemos usar “retales” de código que nos ahorraran tiempo y desgaste de teclado. Hay que ver el video de demostración para entender su poder.

    FuzzyFinder: Efectua búsquedas de todo tipo, archivos, directorios, funciones, etc. Hay que mapear los comandos para adaptarlos a nuestro vimrc. Una vez te acostumbras se hace imprescindible. Con él, el plugin TagList sobra.

    OmniCppComplete: El plugin definitivo para programar en C y C++, dota a Vim de autocompletado para todo tipo de objetos.

    EnhCommentify: Con este plugin podremos facilitar el manejo de comentarios en muy diversos tipos de lenguajes y formatos.

    Plugins para vim

    David García

    2010-02-20 18:46

    NERDTree: plugin para visualizar el sistema de archivos. Interesante mapear F2 por ejemplo al comando provisto por el plugin: NERDTreeToggle. Con esto podemos abrir y cerrar la ventana correspondiente al navegador de archivos.

    TagList: Otro ‘must have’. Con el obtenemos una introspección de clases, funciones, objetos, etc, de los archivos que tengamos abiertos. Interesante también mapear F3 por ejemplo a TlistToggle.

    Surround: Viene muy bien para trabajar con tags. Añade, quita o cambia símbolos. Con un solo comando cs”) pasamos de “palabra” a un (palabra) por ejemplo.

    Repeat: Es un pequeño plugin complementario de Surround para efectuar repeticiones.

    a: Si, solo ‘a’. Este plugin nos va a facilitar la apertura de archivos fuente/cabecera relacionados. Bastante útil para programar en C y C++.

    matchit: Es un % pero supervitaminado, cantidad de mejoras y opciones. Al parecer, desde la versión 6 de Vim se incluye en la distribución estándar.

    vcscommand: Soporte para CVS, Subversion, GIT, SVK, Mercurial y Bazar. Poco más que añadir.

    Algunos comandos 'g' interesantes de vim

    David García

    2010-02-19 17:38

    gd – ir a la definición de la palabra en el cursor

    gD – igual que ‘gd’ pero siempre en el archivo actual

    g< – mostrar la salida del comando anterior

    g? – activar ROT13

    g?? – aplicar ROT13 sobre la linea

    gJ – unir lineas sin meter un espacio en blanco

    ga – mostrar el valor ASCII de la letra bajo el cursor

    gf – editar el archivo con nombre de archivo bajo el cursor

    gv – volver a seleccionar el mismo area seleccionador en el modo Visual
