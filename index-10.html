    Jailbreaking Apple TV 2 with Seas0nPass (...and overcoming 1600 error)

    David García

    2012-05-25 18:44

    Firecore just released an untethered version of Seas0nPass (it’s free) for Apple TV 2d Generation. Previously only tethered version worked which might be a bother if the device reboots. Thanks of course to the long awaited pod2g exploit release.

    I downloaded the last version and stuck with some issues during the jailbreaking process. This is the infamous 160x iTunes family errors (specifically 1600 and 1601). 

    So, after some tryings and restoring processes I managed to jailbreak my device. Here is the process I did:

    - First: If you got any iDevices (iPhones, iPads…) plugged or with Wireless synchronization on, unplug them or put them on flight mode.

    - Get your AppleTV 2 and remote control with you, unplug power and HDMI cables and connect your micro-usb to your Mac/Windows. DON’T PLUG USB ON APPLE TV YET!

    - After download and execute Seas0nPass, click on “Create IPSW”:

    - This will take a while as it is downloading a big .ipsw image. After downloading Seas0nPass will cook the image and when it get ready it will show you this message:

    - Now PLUG THE USB TO YOUR APPLE TV. You will see the little white led blinking SLOWLY so don’t press the buttons yet. When it gets blinking FASTER press the buttons!

    - After a few seconds (7 aprox.) you will see the light blinking SLOWLY again. Time to release the buttons and DON’T TOUCH ANYTHING. Just cross your fingers and let iTunes end the process.

    Ok, the jailbreak should work seamless now.

    I had to admit that this is some cargo cult as the real reason why it doesn’t work remains unknown for me. The key happens when entering DFU mode. If the synchronization is not exactly as above…iTunes shows a 1600 error and stop the process. 1600 errors are concerned to the device communication between iTunes, so I think when DFU mode is not running fine it gets out.

    Hope your jailbreaking works!   

    Easy Wireless Access Point in Windows 7

    David García

    2012-05-17 10:31

    First, run a cmd with Administrator privileges and type this:

    [sourcecode]netsh wlan set hostednetwork mode=allow ssid=yourchoice key=againyourchoice[/sourcecode]

    You can set ‘ssid’ and ‘key’ to whatever fits to you.

    Next let’s start the AP:

    [sourcecode]netsh wlan start hostednetwork[/sourcecode]

    When you’re finish:

    [sourcecode]netsh wlan stop hostednetwork[/sourcecode]

    ‘netsh’ seems to be a very useful tool with a miriad of options. Would be worth to take time to explore it.

    Uninstall all installed apps in MacPorts

    David García

    2012-03-10 22:05

    Sometimes you’ll find that your MacPorts cache is full of trash and is more easy to wipe it all and reinstall your truly needed applications than get rid one by one. So to destroy all MacPorts just write it in a console:

    sudo port -fp uninstall –follow-dependents installed

    Disabling Spotlight in Mac OS X Lion

    David García

    2011-12-09 20:50

    My MacBook Pro is getting older so in order to save some CPU cycles (so I can run more and more VM machines) I managed to find a command able to disable SpotLight Indexing Service. Frankly, I don’t use Spotlight after all, instead I just have a shortcut or launch Launchpad for the apps more frequently used.

    To disable indexing in Spotlight type this in a terminal:

    $ sudo mdutil -i off

    If you got more than one volume you can add: -a

    If you regret and want to get back Spotlight, just:

    $ sudo mdutil -a -i on

    Folding in vim, basic setup

    David García

    2011-11-15 22:35

    Ok, I used a very basic setup for folding. Just put ‘set foldmethod=marker’ in my .vimrc and leave the marker as is set by default, this is: {{{ for start a fold and }}} for end it.

    I haven’t any other set, so it just works fine but I need to “manually” set the folds. A little price I don’t mind to pay.

    Well, let’s say you got a big class on Python but you want to hidden in a fold.

    - Select the whole body class with ‘V}’
    - put :fold

    And that’s all. You got a fold.

    The rest is the usually commands on folding:

    zo, zc open and close a fold
    zO, zC open/close all folds
    zR, zM same as former but sets fold level to highest/0 level

    Repeating a macro in a given range (vim)

    David García

    2011-11-14 15:50

    As someone commented on Vim subreddit, there is a way to apply a macro in a given range, either a mark, lines, visual…

    So, the ‘hack’ is using the ‘g’ command matching all lines (beware, this could break some combinations on macro that includes line up/down movements) and execute a normal command:

    {range}g/./norm @a

    (register ‘a’ used in the example)
