    vim: The same vimrc on multiple computers

    David García

    2012-08-29 22:02

    Reading this post I remember that I never commented here the way I (and a lot of people) use to share the same vimrc on different computers. Instead of doing it as that post says (which is not a great idea because an ‘ack’ may exists in OSX systems via port or homebrew) you can actually differentiate computers by…its name!…surprised?

    Vim has a handy function called ‘hostname’. So, easy, just store it in a variable and let the music go:

    let computername=hostname()

    Now you can adapt your preferences between your computers. For example you can have a laptop with a little screen and a big iMac with a stunning 27 inch display. Obviously your columns and lines would not be the same.

    When we need to substitute an unicode character via a :s// command.

    First we need to know which code the character have, it is easy with just typing: ga

    ga brings us the ascii, hex and octal code of the character.

    next we put it in our s command:

    :%:s/\%u<code>/<substitution/g

    note the % character before the code

    Moving selected text in vim

    David García

    2012-08-22 08:49

    One of the features of other editors (put: SublimeText or Notepad++) that is not native to Vim is move selected text via Ctrl+shift+arrow.

    But of course this is not problem in Vim, just add this to your .vimrc:

    [sourcecode]

    nnoremap <c-s-down> :m+<CR>==
    nnoremap <c-s-up> :m-2<CR>==
    inoremap <c-s-down> <Esc>:m+<CR>==gi
    inoremap <c-s-up> <Esc>:m-2<CR>==gi
    vnoremap <c-s-down> :m’>+<CR>gv=gv
    vnoremap <c-s-up> :m-2<CR>gv=gv

    [/sourcecode]

    source: http://vim.wikia.com/wiki/Moving_lines_up_or_down

    Designated Initializers in Objective C

    David García

    2012-06-30 06:59

    Object construction in Objective C is a two phase procedure. First the object has to be allocated in memory, hence the ‘alloc’ message we always see when a object is invoked this way (and not for example via a ‘get…’, which would be a Factory):

    [sourcecode language="objc"]
    MyObject* o = [[MyObject alloc] init];
    [/sourcecode]

    Second phase involves object initialization which is quite similar to what would be a constructor in C++. When we implement method ‘init’ (which is just a convention) we have to take care on superclass designated initializer . But, what is a designated initializer at all?

    Designated initializer is the method that best set up our object between all initializer methods. An example:

    [sourcecode language="objc"]
    - (id) init:
    - (id) initWithColor:(NSColor*)color;
    - (id) initWithColor:(NSColor*)color andSize:(NSInteger)size;
    [/sourcecode]

    What method do you think is the most complete?

    Our third method includes two parameter while the other only one or none.

    Now, we got a pattern that you are going to see every time you implement your own objects:

    [sourcecode language="objc"]
    - (id) init
    {
    if (self = [super init]) {
    }
    return self;
    }
    [/sourcecode]

    This is the most basic method. It neither do variable initialization nor call other initializers. It just call its superclass initializer, check its got a valid pointer back (thus not nil) and return self.

    But if we got any other initializer methods we must call our most complete initializer. This is the designated initializer. So instead of our light ‘init’ tiny method we would have to call initWithColor:andSize:

    [sourcecode language="objc"]
    - (id) init
    {
    NSColor* color = [NSColor greenColor];
    NSInteger size = 30;
    return [self initWithColor:color andSize:size];
    }
    [/sourcecode]

    and in our initWithColor:andSize:

    [sourcecode language="objc"]
    - (id) initWithColor:(NSColor*)color andSize:(NSInteger)size
    {
    if (self = [super init]) {
    _color = [color retain];
    _size = size;
    }
    return self
    }
    [/sourcecode]

    Protocols in Objective C

    David García

    2012-06-29 12:28

    Just a reference note on how to implement a “protocol” in Objective C.

    First. We define our protocol in a file named “Protocol.h”:

    [sourcecode language="objc"]
    @protocol Protocol

    @required
    - (void)requiredMethod:(NSString*)param;

    @optional
    - (void)optionalMethod;

    @end
    [/sourcecode]

    We can declare a method as optional or required. If the method is required you must implement that method in your class.

    Next, we decided to use the protocol in our class, first the header “UseProtocol.h” then the implementation “UseProtocol.m”:

    [sourcecode language="objc"]
    #import <Foundation/Foundation.h>
    #import "Protocol.h"

    @interface UseProtocol : NSObject <Protocol>
    @end
    [/sourcecode]

     

    [sourcecode language="objc"]

    #import "UseProtocol.h"

    @implementation UseProtocol

    - (void)requiredMethod:(NSString*)param {
    NSLog(@"%@", param);
    }

    @end

    [/sourcecode]

    and next we glue it up together in a main.m

    [sourcecode language="objc"]
    #import <Foundation/Foundation.h>
    #import "UseProtocol.h"

    int main(int argc, const char* argv[]) {
    NSAutoreleasePool* pool = [[NSAutoreleasePool alloc] init];
    UseProtocol* p = [[UseProtocol alloc] init];
    [p requiredMethod:@"test"];
    [p release];
    [pool drain];
    return 0;
    }
    [/sourcecode]

    We could use XCode but is too heavy for our little task…

    If you used an editor, you can compiled it through command line:

    [sourcecode]clang -framework Foundation UseProtocol.m main.m -o main[/sourcecode]

    Adding Ringtones to iOS 5.1 (Jailbreak)

    David García

    2012-05-27 06:15

    As we may expect Apple does the easy almost impossible but…in a elegant way!

    So an easy way to push ringtones to a jailbroken iPhone (iOS 5.1) is (you should have installed ssh services):

    1.- Take your favorite tune and open it in iTunes

    2.- Right click and export to AAC

    3.- Show it in Finder and change file extension from ‘.m4a’ to ‘.m4r’ and drag it to the Desktop (or another handy folder)

    4.- Open a terminal an ssh to your device

    5.- Locate this folder: “/var/stash/Ringtones.aOqYcf/” . It may seem the same except the latter part.

    6.- Copy that path

    7.- Open a terminal in your Mac (or PuTTY in Windows)

    8.- Type this replacing the values:

    scp TONENAME.m4r root@IPOFYOURDEVICE:/var/stash/Ringtones.aOqYcf/TONENAME.m4r

    9.- Enjoy your tune!
