    Gnome: Deshacernos del panel principal

    David García

    2010-06-01 17:20

    Comments

    Si tenemos instalado y tuneado AWN (Avant Window Navigator) y queremos quitar el panel principal, veremos que no se puede ya que está bloqueado para prevenir su borrado accidental.

    Para que el sistema adopte a AWN en vez de Gnome-Panel accedemos al editor de configuraciones del escritorio GNOME.

    gconf-editor

    Buscamos la rama Desktop -> Gnome -> Session -> required_components

    En la entrada panel cambiamos el valor que esté por avant-window-navigator

    vim: Invocar la ayuda desde el editor

    David García

    2010-05-31 19:34

    Comments

    Con el comando K podemos invocar la ayuda asociada al tipo de archivo. Si estamos editando un archivo fuente en C y nos situamos por ejemplo en la función “printf”, saldrá la página man asociada a esa función.

    Lo mismo en Perl o Python. Según la variable :set keywordprg esté asociada al lector de documentación del lenguaje en el que el fichero esté definido.

    En el caso por defecto keywordprg=man buscará en las páginas man. Si queremos que busque en una sección en concreto podemos añadir el número, por ejemplo: 3K mirará en la sección 3 de las páginas man.

    #include

    int main()

        (*p)();

        return 0;

    }

    vim: Movimiento (I)

    David García

    2010-05-10 20:57

    Comments

    Algunas combinaciones y comandos para movernos por la pantalla:

    h,j,k,l -> izquierda, abajo, arriba, derecha

    Ctrt + e -> desplazar pantalla hacia abajo
    Ctrl + y -> desplazar pantalla hacia arriba

    H, M, L -> Arriba (High), Mitad (Middle), Abajo (Low)

    Ctrl + f -> Avance página
    Ctrl + b -> Retroceso página

    ma -> Poner una marca (a)
    ‘a -> Ir a la marca (a)

    gg -> Principio
    G -> Final

    [numero_de_linea]gg
    [numero_de_linea]G Ir a la linea [numero_de_linea]
    :[numero_de_linea]

    w,W -> Avanzar por palabra (sin/con símbolos)
    b,B -> Retroceder por palabra (sin/con símbolos)
    e,E -> Ir al final de la palabra (sin/con símbolos)

    f[caracter] -> ir al carácter [carácter] más próximo
    F[carácter] -> ir al carácter [carácter] más próximo

    ‘. -> Ir a la última linea editada

    Ctrl + o -> Ir a la posición anterior del cursor
    Ctrl + i -> Ir a la posición posterior del cursor

    vim: Filtrar texto (varios trucos)

    David García

    2010-04-28 21:08

    Comments

    * Antes una utilísima función para “grepear” partes del texto hacia un registro:

    :g/patron/y A

    Se guardará en el registro “a

    Básicamente es: g = grep /patron_a_encontrar/ y = yank A = append

    * Leer la salida de un comando hacia el editor:

    :r!ls

    r = read ! = ejecutar_un_comando ls = comando_ls

    :-r!ls

    Lo mismo que el anterior pero el caracter ‘-’ al principio hará que el texto se inserte antes de la linea actual.

    * Usar un programa externo para filtrar texto:

    :%!sort

    % = todo_el_fichero ! = ejecutar_un_comando sort = comando_sort

    :10,20!sort

    Lo mismo que el anterior pero solo de la linea 10 a la 20

    V
    (seleccionar lineas)
    !sort

    Es lo mismo que los anteriores pero usando el modo visual linea

    vim: Como ver símbolos ocultos

    David García

    2010-04-25 09:49

    Comments

    Para hacer que Vim muestre los símbolos ocultos (saltos de linea, etc), tan solo tenemos que introducir:

    :set invlist

    Para volver a la situación anterior igualmente introducimos el mismo comando.

    La instrucción en si es ‘list’ que mostrará los caracteres ocultos, sólo que con ‘inv’ actuará de interruptor lógico, inviertiendo el estado actual.

    vim: deshabilitar "preview" en omni-completion

    David García

    2010-04-12 06:40

    Comments

    A veces resulta molesto cuando activamos omni-completion y nos sale la ventana de ‘preview’.

    Para quitarla podemos añadir en nuestro .vimrc la siguiente línea:

    set cot -= preview

    cot es la abreviatura de complete options. Si queremos saber más :h cot.
