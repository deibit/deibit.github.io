    vim: el plugin TagList en Mac OS X

    David García

    2010-09-30 13:04

    El muy útil plugin para Vim TagList necesita de ‘Exhuberant tags’ para funcionar. El ctags que trae Mac cuando se instala XCode y compañía presenta ciertos problemas cuando es usado por TagList (no funciona correctamente).

    Lo primero es instalar ‘ctags’ desde MacPort:

    sudo port install ctags

    Tras la instalación debemos incorporar a nuestro .vimrc la siguiente cadena:

    let Tlist_Ctags_Cmd=’/opt/local/bin/ctags’

    Con esto le decimos al plugin que cuando quiera usar ctags éste se encuentra en la localización que le hemos añadido. Así podrá usar el ctags de MacPort que es el que necesita.

    vim, saltar por el texto "ajustado"

    David García

    2010-09-29 20:13

    Para que Vim nos deje pasar el cursor por lineas que están ajustadas (lineas que sobrepasan el número de columnas), podemos poner en nuestro vimrc:

    nmap gj
    nmap gk
    nmap j gj
    nmap k gk

    Los dos primeros son para los cursores, los otros dos para cuando usamos el “modo puro”.

    Python: Incoherencias en los nombres

    David García

    2010-07-09 00:44

    Una de las cosas que más valoro de Java es la consistencia en el empleo de estilo con respecto a los nombres de métodos, clases, etc. En Python, a pesar del PEP8 que indica o recomienda ciertos usos, nos encontramos con una variedad de estilos en los nombres de funciones y métodos de clases:

    Del módulo ‘threading‘:

    Como se inspiró en la API de Java, nos encontramos con el camelCase habitual:

    threading.currentThread()

    Sin embargo, han sido renombrados para que sean conformes al PEP8:

    threading.current_thread()

    y además: “However, no schedule has been set for the deprecation of the camelCase names and they remain fully supported in both Python 2.x and 3.x.”

    Lo cual se traduce que incluso en la versión 3 de Python nos vamos a topar con distintos estilos en los fuentes que encontremos.

    Más. Los nombres de función han de ser en minúsculas y separados por guión bajo:

    esto_es_una_funcion()

    Pero, nuevamente en el módulo ‘threading‘:

    threading.Condition()
    A factory function that returns a new….

    No acaba aquí la cosa, en los nombres de módulos:

    Modules should have short, all-lowercase names.

    Y como no, nos encontramos en los módulos estándar:

    BaseHTTPServer
    ConfigParser
    cPickle
    cProfile
    …

    Aunque muchos de ellos se están pasando al estilo del PEP8.

    Esto (y el cansino ‘self’) es una de las cosillas que molestan de Python. Seguir un estilo es utilísimo, con sólo un vistazo, sabemos que tenemos delante, máxime en un lenguaje dinámico donde no tenemos el apoyo de la sintaxis explicita (int, float…) al declarar un objeto.

    vim: comandos (no tan) básicos

    David García

    2010-06-29 06:46

    vim -O archivo1 archivo2

    Abre dos archivos en la misma ventana.

    :ba

    Abre una ventana por cada bufer.

    ctrl+g

    Información del archivo que estamos editando.

    zt

    Hace scroll hasta la linea actual del cursor.

    “_x

    Borra el texto seleccionado pero no lo deposita en ningún registro.

    “Ay

    Anexa el texto seleccionado en el registro “a”.

    :%!sort

    Filtra el bufer entero a través de un comando externo.

    :,!comando

    Sustituye la linea actual por la salida del comando.

    vim: Corrector ortográfico

    David García

    2010-06-02 22:38

    Para añadir corrección ortográfica a VIM sólo tenemos que introducir:

    :setlocal spell spelllang=es

    Podemos escribirla en el .vimrc para que esté activada por defecto y añadiendo junto a esta opción las siguientes:

    runtime plugins/spellfile.vim
    set spellfile=~/.vim/dict.add

    Esto nos permite mantener un archivo a modo de diccionario de palabras que no estén en el archivo ortográfico y queramos ir añadiendo.

    Al recargar el .vimrc o arrancar Vim nos preguntaŕa si queremos descargar el archivo de corrección ortográfica y otras cuestiones. Con las respuestas por defecto valdrá para una instalación estándar.

    Funcionamiento

    Vim señalará en rojo (si Vim fue compilado con syntax habilitado) las palabras que encuentre con errores ortográficos. En el modo normal disponemos de ‘z=’ para poder ver las opciones en un búfer.

    En el modo de inserción podemos hacerlo con el autocompletado Ctrl + x s. Nos presentaŕa un menú con las distintas opciones.

    Deshabilitar la corrección

    Si nos resulta molesta la corrección por cualquier motivo podemos definir una combinación de teclas para deshabilitarla o directamente :nospell desde ex.

    En el .vimrc:

    nmap ns :set nospell

    vim: Cambiar el tamaño de las ventanas

    David García

    2010-06-01 19:43

    En Vim podemos partir una ventana en dos tanto vertical como horizontalmente:

    Ctrl + w v

    Ctrl + w s

    Para poder modificar su tamaño con respecto de las demás:

    Añadir/Restar tamaño [n] vertical:

    [n] Ctrl + w +

    [n] Ctrl + w  -

    Añadir/Restar tamaño [n] horizontal:

    [n] Ctrl + w <
    [n] Ctrl + w >

    Establecer un tamaño igual para todas las ventanas (si es posible):

    Ctrl + w =
