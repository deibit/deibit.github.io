    C++: Puntero a función miembro de un objeto

    David García

    2010-10-27 15:19

    En C++ si queremos tener un puntero a la función miembro de un objeto no podemos hacerlo como si fuese C, es decir:

    typedef void (*funcion) ();
    funcion = &t::f_Miembro;

    En lugar de ello la sintaxis es algo más curiosa:

    typedef void (Test::*pMember)(void);
    pMember p;

    Luego, para ir asignando las funciones miembro:

    p = &Test::Member;
    …
    p = &Test::Member2;

    Es decir, cada vez que asignemos una función miembro y al igual que en la definición se hace
    indicando la clase. Luego para usar ese puntero si se referencia al objeto en particular:

    Test *t = new Test();
    (t->*p)();

    Es decir, no podemos directamente “atar” una función miembro de un objeto en particular a nuestro puntero y posteriormente usarla llamando: p();

    El puntero prácticamente se usa para seleccionar una función del objeto y ser usada a través del objeto. En tiempo de ejecución puede ser bastante útil para usos de plugins, etc.

    El código:

    [sourcecode language="cpp"]
    #include <iostream>

    class Test
    {
    public:

    void Member(void)
    {
    std::cout << "test" << std::endl;
    }

    void Member2(void)
    {
    std::cout << "test2" << std::endl;
    }
    };
    typedef void (Test::*pMember) (void);
    int main()
    {
    Test *t = new Test();
    pMember p = &Test::Member;
    (t->*p)();
    p = &Test::Member2;
    (t->*p)();
    return 0;
    }
    [/sourcecode]

    vim: cambiar la codificación de archivos

    David García

    2010-10-18 06:11

    En bash, con el comando ‘file’ podemos averiguar que tipo de archivo (codificación, tipo, etc) vamos a editar.

    Recordad que esto lo podemos hace directamente en Vim :!file

    Para cambiar la codificación actual usamos :set fileencoding=

    En mi caso, con un vimrc que sincronizo entre las máquinas (un windows, varios mac y linux) cada vez que edito el .vimrc en una cambio sin querer la codificación y determinados mapeos no se procesan adecuadamente en según que sistema. Por ejemplo al mapear la ‘ñ’ en windows.

    Para que en Mac y Linux puedan procesar un vimrc editado en la codificación que usa Windows:

    :set fileencoding=utf-8

    y :wq

    vim: Convertir archivos de DOS a UNIX

    David García

    2010-10-09 19:48

    Al pasar archivos editados en VIM de un sistema a otro puede ocurrir que nos de error cuando son interpretado (las dichosas terminaciones entre otros).

    Esta secuencia Ex hace que se guarde el archivo del bufer actual de DOS a UNIX:

    :update
    :e ++ff=dos
    :setlocal ff=unix
    :w

    Actualiza el bufer actual. Añade a ‘ff’ (file format) el formato dos, indicando que lo así sea interpretado. Ajusta ‘ff’ al valor unix y convierte el archivo. Por último guardamos.

    Visto y más información aquí.
