    Solution to XBMC crashes on Apple TV2

    David García

    2011-10-14 23:49

    Comments

    I was watching a tv show when xbmc suddenly crashes Apple TV. It rebooted and I continued play the movie. 10 minutes later the issue appeared again…and so on but always a 10 minutes lapse.

    I checked manually if a new update for ATV2 would exists and yes, there was a new update. I pressed later (thanks, but I want my device as is it) and figured out how I could cut ATV2 from calling back home.

    Easy, tell ATV2 a nice 127.0.0.1 answer to its request when asking for the update center domain.

    ¿What’s the domain?

    As a programmer I’m soooo lazy. One way would be listening tcpdump for a DNS request, but this is a large amount of time (10 minutes) and a lot of keystroking.

    Gooling a little I found the domain.

    ssh your ATV2 (root@IP)
    apt-get install nano
    nano /etc/hosts

    and add this:

    127.0.0.1 mesu.apple.com

    After watching 2 shows more XBMC is still running and waiting orders!

    Pointing sqlmap obfuscated parameters in URLs

    David García

    2011-09-23 08:35

    Comments

    Well I called it obfuscated parameters albeit what happens is that the params are rewritten, let’s see:

    This is a common URL:

    http://www.prisonernumbersix.com/article.php?id=1231

    But we can rewrite that URL and show an alternative, more beautiful:

    http://www.prisonernumbersix.com/article/1231

    sqlmap works testing every parameter by default, optionally we can point a parameter with ‘-p’ option.

    Problem here is that sqlmap is unaware when we try to test a “beautiful” URL.

    Solution is easy: mark parameter as an *

    Taking our example URL we call sqlmap this way:

    ./sqlmap.py -u”http://www.prisonernumbersix.com/article/param1*” and that’s all!

    Two handy functions for turning on-off an 'IDE' in vim

    David García

    2011-09-13 11:10

    Comments

    Sometimes I found myself working on Vim as a simple editor, just to write a single file (tweaking a configuration file and similar stuff) but frequently I’m working on a project, these kind of project you’d use a complete IDE (put here Eclipse, NetBeans…).

    Vim fills my needs in both sides so I worked two functions that make switching from side to side in a flick. Just put this in your vimrc and change key triggers as you want:

     1 nmap &lt;leader&gt;ide :call GetIDE()&lt;CR&gt;
 2 function! GetIDE()
 3     if !exists(&quot;*GetIDE&quot;)
 4         return
 5     endif
 6     NERDTree
 7     TagbarOpen
 8     set columns +=20
 9 endfunc
10 
11 nmap &lt;leader&gt;edi :call GetEDI()&lt;CR&gt;
12 function! GetEDI()
13     if !exists(&quot;*GetEDI&quot;)
14         return
15     endif
16     NERDTreeClose
17     TagbarClose
18     set columns -=20
19 endfunc

    Just the idea is simple: leader+ide to get a full flavor IDE and leader+edi to return to an EDItor (pun wasn’t intended when named the function EDI = reverse(IDE);)

    Code still needed a review and maybe an improvent but works right. Of course you’ll need NERDTree and Tagbar plugins but is easy to change these for your favorite plugin.

    Hope it will be useful for you.

    Como funcionan los registros de vim

    David García

    2011-09-08 23:02

    Comments

    Copiar, pegar y cortar son conceptos que en Vim pueden llegar a ser un poco liosos si no se aprende bien como funciona el tema de los registros.

    Los registros son “espacios” donde Vim guarda aquello que copiamos (yank) o borramos (delete). Cualquier operación que signifique borrar, copiar o en definitiva cambiar el texto va a ser “registrada” por Vim.

    Vim posee 9 de estos registros, vamos a ver para que y como funciona cada uno de ellos:

    El registro sin nombre

    Invocado como “”

    Vim copia aquí todo lo que se borra o copia. También copia a este registro aquello que se ha escrito en el último registro referenciado. Es decir:

    “ay3w (seleccionar registro ‘a’ y copiar 3 palabras)

    Podremos comprobar como tanto en el registro ‘a’ y en el registro sin nombre o ‘”‘ se halla el mismo valor.

    El registro sin nombre es como un puntero a la última acción efectuada sobre un registro.

    Los registros numerados del 0 al 9

    El registro ’0′ se usa cada cada vez que se copia. Todo lo que copiemos estará en el registro ’0′.

    Del 1 al 9 son registros que van a ir secuencialmente todo aquello que cambiemos en el texto. Cada cambio situará el texto contenido en un registro al siguiente (como una cola) al salir del registro ’9′ el contenido se perderá.

    El registro de borrado o ‘-’

    Aquí se sitúan los datos borrados siempre que el comando haya afectado a menos de una linea y no se haya invocado un registro específicamente.

    Registros nombrados o registros a-z y A-Z

    Estos son los registros que el usuario puede invocar. Si se invoca como letra minúscula el contenido anterior se perderá. Si se invoca en mayúsculas el contenido será agregado al contenido anterior que tuviera el registro.

    Registros de solo lectura ‘.’ ‘%’ ‘#’ y ‘:’

    “. este registro contendrá el último texto insertado. Un registro tremendamente útil cuando se realizan operación de inserción repetitivas.

    “% Contiene el nombre del archivo

    “# Contiene el nombre del archivo alternativo

    “: Contiene el último comando usado. Por ejemplo podemos hacer @: para volver a ejecutar el último comando otra vez.

    El registro de expresión “=

    Al invocar este registro Vim nos sitúa en la linea de comandos donde podremos introducir cualquier expresión válida que se evalúe o convierta a una cadena.

    Los registros de selección y soltar (drag’n'drop) “* “+ y “~

    Estos registros son usados cuando se usa Vim en ambientes GUI. Escritorio de Windows, Mac OSX y X11.
